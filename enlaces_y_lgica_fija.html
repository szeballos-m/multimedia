Enlaces y Lógica Fija

En esta unidad, pasamos del diseño visual a la funcionalidad. La lógica de navegación es el motor que permite que un sistema deje de ser una serie de imágenes estáticas y se convierta en una experiencia interactiva real.

 

Desarrollo de la lógica de navegación clara y coherente

La lógica de navegación es el conjunto de reglas y estructuras que determinan cómo el usuario se desplaza por el sistema multimedia. Una navegación clara implica que el usuario siempre sepa dónde está, cómo volver atrás y cómo avanzar. Se recomienda utilizar mapas de navegación previos, botones consistentes y rutas bien definidas entre escenas o pantallas. La coherencia en la navegación mejora la experiencia de usuario y reduce la confusión.

Este conjunto de reglas determinan cómo y cuándo el usuario puede desplazarse por el sistema. No se trata solo de poner botones, sino de gestionar el flujo.

  • Navegación Libre: El usuario puede ir a cualquier parte en cualquier momento (común en menús principales).
  • Navegación Restringida/Lineal: El usuario debe cumplir una acción (ej. ver un video completo o responder una pregunta) antes de que el botón "Siguiente" se habilite.
  • Navegación Condicional: El destino del usuario cambia según sus acciones previas (ej. si el usuario eligió "Nivel Principiante", el hipervínculo lo lleva a una escena distinta que si eligió "Experto").

Programación de hipervínculos absolutos y relativos

Los hipervínculos permiten conectar diferentes partes del sistema o acceder a recursos externos.

Los enlaces absolutos incluyen la ruta completa (por ejemplo, 'https://sitio.com/pagina'), útiles para acceder a contenido en línea.

Los enlaces relativos se refieren a archivos dentro del mismo proyecto (por ejemplo, '../imagenes/foto.png'), ideales para sistemas locales o empaquetados. Es fundamental elegir el tipo de enlace adecuado según el entorno de ejecución del sistema.

En sistemas multimedia, un hipervínculo es un evento de escucha (EventListener) que ejecuta una acción de salto. Debemos distinguir dos tipos según el destino:

Hipervínculos Relativos (Internos)

Son los que conectan partes del mismo proyecto. Se mueven a través de la línea de tiempo o escenas.

  • Uso: Navegar del Menú a la Galería, o volver al Inicio.
  • Ejemplo en JavaScript (Animate):  

this.btn_contacto.addEventListener("click", function() { 

   this.gotoAndStop("escena_contacto");

}.bind(this));

Hipervínculos Absolutos (Externos)

Son los que dirigen al usuario fuera del sistema, generalmente a una URL en internet.

  • Uso: Redes sociales del cliente, descarga de un PDF externo o bibliografía web.
  • Ejemplo en JavaScript: 

this.btn_web.addEventListener("click", function() { 

   window.open("https://www.google.com", "_blank");

});

 Manejo básico de variables

Las variables permiten almacenar información temporal durante la navegación del usuario.

Un contador de secciones vistas puede incrementarse cada vez que el usuario accede a una nueva pantalla, lo que permite desbloquear contenido o mostrar estadísticas. Asimismo, se pueden usar variables para controlar la visibilidad de un menú de ayuda, mostrando u ocultando información contextual según la interacción del usuario. Estas funcionalidades enriquecen la interactividad del sistema y personalizan la experiencia.

Caso A: Contador de Secciones Vistas

Sirve para saber si el usuario exploró todo el contenido antes de permitirle finalizar.

  1. Declaración: let seccionesVistas = 0;
  2. Acción: Cada vez que el usuario entra a una sección nueva, sumamos: seccionesVistas++;
  3. Lógica: if (seccionesVistas >= 5) { habilitarBotonFinal(); }

Caso B: Menú de Ayuda o Estados (Toggles)

Podemos usar variables booleanas (true/false) para controlar elementos de la interfaz, como un panel de ayuda que se abre y cierra con el mismo botón.

Lógica:

Antes de que el usuario haga nada, necesitamos definir cómo empieza el sistema.

  • Creamos la variable ayudaVisible y la ponemos en false.

Esto le dice al sistema: "El panel de ayuda está cerrado".

  • Cuando el usuario hace clic, el programa primero hace una pregunta:

"¿Es verdad que la ayuda NO se está viendo?" (if ayudaVisible == false)

Las dos rutas posibles (If / Else)

  • (Si es Falso): el panel estaba oculto, el código lo hace visible (visible = true) y actualiza la memoria: ahora ayudaVisible pasa a ser true.
  • (Si es Verdadero): el usuario hace clic otra vez, el sistema vuelve a preguntar. Como ahora la variable es true, el código salta directamente al else (el "si no..."). Entonces, oculta el panel y vuelve a poner la variable en false.

JavaScript

let ayudaVisible = false;

 this.btn_ayuda.addEventListener("click", function() { 

   if (ayudaVisible == false) { 

       this.panel_ayuda.visible = true; 

       ayudaVisible = true;  

  } else { 

       this.panel_ayuda.visible = false;

        ayudaVisible = false;

    }

}.bind(this));

 

Nota: La combinación de variables (memoria) con hipervínculos (acción) es lo que define la "Lógica Fija" de un sistema cerrado. Un buen desarrollador multimedia siempre prevé qué pasa si el usuario intenta saltarse pasos.
