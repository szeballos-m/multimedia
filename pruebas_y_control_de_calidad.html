Pruebas y Control de Calidad

Aseguramiento de la Calidad (QA) en sistemas multimediales cerrados

Metodologías de Prueba

En sistemas cerrados (kioscos, stands), el entorno es controlado, pero el usuario no. Las pruebas se dividen en:

  • Testing de Ruta (Path Testing): Consiste en recorrer todas las combinaciones posibles de navegación. Si el sistema tiene 10 escenas, el tester debe asegurar que se puede llegar a la 10 desde todas las rutas lógicas previstas.
  • Pruebas de Usabilidad: Observación de usuarios reales interactuando con el sistema sin ayuda. Se busca detectar si los botones son claros (affordances) y si el flujo es intuitivo.

Pruebas y Calidad:

  • Stress Testing: ¿Qué pasa si el usuario toca la pantalla 50 veces seguidas o deja el sistema encendido 48 horas? (Crucial para sistemas cerrados en museos o stands).
  • Protocolo de Reporte: Cómo escribir un reporte de errores (Bug Report) que un programador pueda entender (Pasos para reproducir, resultado esperado vs. resultado obtenido).

Detección y Corrección de Errores

  • Dead-ends (Callejones sin salida): Escenas donde el usuario llega y no tiene un botón para volver atrás o seguir adelante.
    • Corrección: Revisar que cada pantalla tenga al menos un hipervínculo de retorno o un botón de "Inicio".
  • Bugs Funcionales: Errores en la lógica (ej. una variable que no suma puntos o un video que no se reproduce).
  • Enlaces Rotos: Botones que apuntan a escenas que fueron borradas o renombradas durante la producción.

Enfoque: Garantizar que el producto no falle en manos del usuario.

  • Metodologías de Prueba:
    • Testing de Ruta (Path Testing): No es solo hacer clic; es validar cada camino posible del mapa de navegación. Se debe crear una "matriz de pruebas" donde se marque cada enlace verificado.
    • Pruebas de Usabilidad (Pasillo): Observar a un tercero (que no conozca el proyecto) usar el sistema. Si duda más de 3 segundos en qué botón apretar, hay un error de diseño.
  • Detección de "Dead-ends" (Callejones sin salida):
    • Identificar pantallas que carecen de botón "Home" o "Atrás".
    • Verificación de botones de cierre: En sistemas cerrados, el usuario no debe poder cerrar la app, pero el administrador sí (mediante una combinación de teclas oculta).
  • Bugs Funcionales:
    • Variables que no se reinician al empezar de nuevo.
    • Videos que se solapan si se aprieta el botón de "Play" muchas veces.
