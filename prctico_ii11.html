Práctico II.1.1

Creación de la Guía de Estilo gráfica

Actividad:

  • Elaborar una Guía de estilo gráfica (ver ANEXO: Guía práctica), que incluya:
    • Paleta cromática coherente con el proyecto.
    • Tipografías principales y secundarias.
    • Set de iconos básicos (navegación, ayuda, acciones).

 Metodología:

Uso de software de diseño gráfico y UI (ej. Figma, Wich Editor, Sketch). Aplicación de principios de diseño visual y consistencia tipográfica.

Entregables:

  • Guía de estilo gráfica documentada.

Objetivo:

Aplicar principios de diseño gráfico y UI a un sistema funcional, asegurando coherencia visual y usabilidad.

Criterios de evaluación:

  • Legibilidad a Distancia: ¿El tamaño de la letra es cómodo para la distancia real a la que estará el usuario de la pantalla?
  • Tamaño de Objetos Táctiles: Si es una pantalla touch, ¿los botones son lo suficientemente grandes (mínimo 44px) para no fallar el toque?
  • Formatos Multimedia: ¿Se especifican los formatos de video y audio (ej. .mp4, .wav) para que el sistema no se trabe?
  • Feedback Visual: ¿El sistema reacciona visualmente cuando el usuario toca algo (cambio de color o forma)?
  • Tiempo de Carga: ¿Hay un diseño definido para cuando el sistema está "cargando" contenido (un spinner o barra)?
  • Jerarquía de Color: ¿El color ayuda a resaltar lo importante o solo distrae del contenido principal?
