Empaquetado y Distribución Final

De la Autoría al Entorno Real: Exportación y Despliegue

Procesos de Compilación

El empaquetado depende de la plataforma de destino:

  • Ejecutables (EXE/APP): Para computadoras sin conexión a internet.
  • HTML5/Web: Para ejecución en navegadores locales.
  • Consideración técnica: Asegurar que todos los activos (videos, fuentes, sonidos) estén vinculados correctamente o embebidos en el paquete final.

Configuración del Entorno de Destino

  • Modo Kiosco: Configuración del sistema operativo para que el usuario no pueda salir de la aplicación (bloqueo de teclas como Alt+Tab o la tecla Windows).
  • Permisos y Seguridad: Asegurar que el hardware (pantalla táctil, sensores) tenga los drivers instalados y permisos de ejecución.
  • Auto-ejecución: Configurar el equipo para que, al encenderse, la aplicación se abra automáticamente sin intervención humana.

La salida del entorno de diseño al entorno de uso real

  • Compilación y Exportación:
    • Configuración de la resolución nativa: Asegurar que el proyecto corra a la resolución exacta del monitor de destino (ej. 1920x1080) para evitar pixelado o bordes negros.
    • Inclusión de librerías: Verificar que el paquete incluya el motor de ejecución (Runtime) para que no dependa de si la computadora tiene software instalado.
  • Entorno de Destino (Modo Kiosco):
    • ¿Qué es?: Configurar el sistema operativo para que la app sea la única interfaz disponible.
    • Permisos: Configuración de "Ejecutar como administrador" para evitar ventanas emergentes del sistema que bloqueen la interacción del usuario.
    • Hardware: Calibración de la sensibilidad táctil si se usa un monitor touch.

Gestión de Assets: Embebidos vs. Vinculados

Diferencia entre recursos "embebidos" y recursos "vinculados" (como videos pesados en 4K que no deben ir dentro del ejecutable).

La decisión de cómo guardar un archivo depende principalmente de su peso y de qué tan seguido necesites cambiarlo.

Recursos Embebidos (Internal Assets)

Son archivos que se "empaquetan" dentro del ejecutable o del archivo principal de la aplicación (como un .exe, .apk o .app).

  • Qué son: Iconos de la interfaz, fuentes, sonidos cortos de UI o scripts.
  • Pros: Seguridad (nadie puede borrarlos por error), rapidez de acceso y facilidad de distribución (un solo archivo lo tiene todo).
  • Contras: Si quieres cambiar una imagen, debes volver a compilar y exportar todo el proyecto. Si el archivo es muy grande, el programa tardará mucho en cargar en la memoria RAM.

Recursos Vinculados (External/Linked Assets)

Son archivos que viven fuera del ejecutable, en carpetas locales o en la nube, y el programa los llama mediante una ruta de acceso (path).

  • Qué son: Videos 4K, galerías de fotos pesadas o bases de datos.
  • Pros: Mantienen el ejecutable ligero. Puedes reemplazar un video de 2GB por otro con el mismo nombre y la aplicación lo mostrará sin necesidad de tocar el código.
  • Contras: Si el usuario mueve o borra la carpeta externa, la aplicación mostrará un error de "archivo no encontrado".

Actualización Remota Básica (Hot-Swapping)

Cómo se actualizaría un contenido sin tener que reinstalar todo el sistema (ej. cambiar un archivo .json o una carpeta de imágenes).

Para actualizar contenido sin reinstalar todo el sistema, se utiliza una arquitectura de separación de datos y lógica. La clave aquí es que el software sea "ciego" y solo lea lo que digan los archivos externos.

El flujo de trabajo típico:

  1. El archivo de configuración (.JSON): En lugar de escribir los textos o nombres de archivos dentro del código, usas un archivo JSON.
    • Ejemplo: El programa lee config.json y busca la clave "version_feria": "2026". Si quieres que diga "2027", solo editas ese texto en el bloc de notas.
  2. Estructura de Carpetas Espejo: Creas una carpeta llamada /Assets_Externos/. El programa está programado para cargar todo lo que esté dentro de esa ruta.
  3. Sincronización:
    • Manual: Sobrescribir los archivos vía USB o FTP.
    • Automatizada: Al iniciar, la app consulta un servidor: "¿Hay una versión nueva del JSON?". Si la respuesta es "Sí", la app descarga el nuevo JSON y las nuevas imágenes a la carpeta local antes de mostrar la interfaz.

Característica

Embebido

Vinculado (Externo)

Uso ideal

Logotipos, sonidos de botones.

Rendimiento

Nula (requiere re-compilar).

Grande y pesado.
