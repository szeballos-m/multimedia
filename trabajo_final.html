Trabajo Final

Proyecto Final: Sistemas Multimedia Cerrados

Contexto: Desarrollo de un Sistema Multimedial Cerrado (Kiosco interactivo / Stand-alone).

1. Desafío

Diseñar y desarrollar un sistema multimedia cerrado e inalterable que resuelva una necesidad de información o interacción en un contexto físico específico. El sistema debe ser autoexplicativo, ya que no contará con un mediador humano.

2. Posibles Temáticas (Elegir una)

  • Museografía Digital: Guía interactiva para una sala de museo específica.
  • Kiosco Informativo: Punto de información para una institución, hospital o terminal de transporte.
  • Terminal de Autogestión: Simulador de trámites o compra de tickets.
  • Experiencia Educativa (Edutainment): Juego o módulo de aprendizaje para stands de ferias de ciencias.

3. Requisitos Técnicos Mínimos

Para que el proyecto sea aprobado, el ejecutable final debe contar con:

  • Navegación: Al menos 3 niveles de profundidad (Home > Categoría > Detalle). Uso de nodos de información fragmentada (un concepto por pantalla).
  • Medios Integrados: Uso obligatorio de texto, gráficos/imagen (bitmap y vectorial), audio (ambiente o locución) y video optimizado.
  • Interactividad: Botones con estados (normal, hover, active), menús de navegación consistentes y al menos un elemento de lógica (ej: contador de progreso o feedback de usuario).
  • Salida: Archivo ejecutable (EXE/APP) o paquete empaquetado para pantalla táctil/totem.

5. Criterios de Evaluación

  • Usabilidad: ¿Puede un usuario nuevo entender el sistema sin ayuda? (Autoexplicación). ¿El usuario se pierde? ¿Sabe siempre cómo volver al inicio? ¿Es fácil volver al inicio o cambiar de sección sin frustración?
  • Arquitectura de Información: Claridad en la jerarquía de los contenidos.
  • Calidad UI: Consistencia visual, legibilidad tipográfica y uso de la paleta de colores. ¿El sistema mantiene el mismo estilo en todas sus pantallas?
  • Optimización: Fluidez de los medios y peso del archivo final. ¿Los enlaces funcionan correctamente? ¿El video y audio están sincronizados?

Checklist de Pre-Entrega: Sistema Cerrado Final

1. Funcionalidad y Navegación (El motor)
  • Cero Callejones Sin Salida: ¿Todos los nodos tienen un botón para volver atrás o al Inicio?
  • Botones Operativos: ¿Todos los botones funcionan y apuntan al destino correcto?
  • Feedback Visual: ¿Los botones cambian de estado (hover / click) al interactuar con ellos?
  • Consistencia de la Interfaz: ¿El menú de navegación principal está siempre en el mismo lugar?
2. Gestión de Medios (La optimización)
  • Peso de Archivos: ¿Los videos están comprimidos o el proyecto tarda más de 3 segundos en cargar una escena?
  • Normalización de Audio: ¿El volumen de la música de fondo y las locuciones es coherente entre sí (nada suena demasiado fuerte o despacio)?
  • Calidad Visual: ¿Las imágenes se ven nítidas o están pixeladas/estiradas?
  • Fuentes Tipográficas: ¿Las fuentes están embebidas o convertidas a curvas para que no se pierdan en otra computadora?
3. Experiencia de Usuario - UX (La empatía)
  • Regla de los 5 Segundos: ¿Un usuario nuevo entiende de qué trata el sistema antes de que pasen 5 segundos?
  • Legibilidad: ¿Hay suficiente contraste entre el texto y el fondo? (Nada de texto gris sobre fondo gris).
  • Jerarquía: ¿Lo más importante de cada pantalla es lo que primero se ve?
4. Empaquetado y Documentación (El cierre)
  • Ejecutable Autónomo: ¿El archivo funciona por sí solo sin necesidad de tener instalado el software de autoría?
  • Nombre de Archivo Profesional: ¿El archivo se llama NombreProyecto_VFinal.exe y no trabajo_final_FINAL_este_si_2.exe?
  • Manual Técnico: ¿El PDF del manual está en la misma carpeta que el ejecutable?
  • Rutas Relativas: ¿El proyecto funciona si lo muevo de la carpeta o de una computadora a otra?

Arquitectura de Información

Estructura lógica, fluida y sin errores de navegación. Mapa y Guion coinciden 100% con el producto.

Estructura clara, pero con saltos bruscos o 1 enlace roto menor.

Navegación confusa. El usuario se pierde ocasionalmente.

Estructura incoherente o múltiples "callejones sin salida".

Diseño UI y Estética

Interfaz profesional. Uso experto de color, tipografía y jerarquía. Estados de botón perfectos.

Interfaz funcional y prolija. Consistencia visual en la mayoría de las pantallas.

Diseño pobre o inconsistente. Falta de contraste o jerarquía visual.

Interfaz ilegible o que dificulta el uso del sistema.

Integración Técnica

Medios (video/audio) perfectamente optimizados, sincronizados y de alta calidad.

Medios funcionales, aunque con detalles menores de optimización o volumen.

Archivos pesados que ralentizan el sistema. Errores de carga ocasionales.

Los medios no cargan, se ven pixelados o no tienen relación con el contenido.

Usabilidad (UX)

El sistema es 100% autoexplicativo. Supera con éxito las pruebas de usuario.

El sistema se entiende bien, requiere mínima curva de aprendizaje.

El usuario requiere ayuda externa para completar tareas básicas.

El sistema es frustrante o imposible de usar sin guía constante.

Documentación (Manual)

Manual técnico completo, profesional y útil para un técnico externo.

Manual completo, pero con omisiones menores en requerimientos de hardware.

Documentación incompleta, desordenada o puramente teórica.

No entrega manual o es una copia de la consigna sin datos reales.
